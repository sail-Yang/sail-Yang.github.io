vscode使用指南

发表于 2021-09-03 更新于 2021-09-09 分类于 工具使用 ， IDE/编辑器 阅读次数： Valine：

前言

最近想学一下vscode,将vscode当作主要使用的编辑器。所以想专门写一篇，来记录一下使用和配置的过程，一开始不会有体系，只是遇到什么问题就记什么，当用得差不多时，可能会回头整理一下。

快捷键

全局快捷键

作用 快捷键
打开/关闭侧边栏 Ctrl+B
打开多个编辑器 Ctrl+\
聚焦到第n个编辑器 Ctrl+n(数字)
放大/缩小 Ctrl + +/-
进入专注模式 进：Ctrl+K再按M,退：Esc twice

删除操作

作用 快捷键
删除整行 Ctrl+Shift+K
删除光标之前的一个英文单词 Ctrl+backspace
删除光标之后的一个英文单词 Ctrl+delete

代码编辑快捷键

作用 快捷键
光标移动到开头/结尾 home/end
光标移动到文档开头/结尾 Ctrl+home/Ctrl+end
回到上一个光标的位置 Ctrl+U
代码上下换位 Alt+↑↓
代码复制并填充上/下 Alt+Shift+↑↓
快捷注释（单行） Ctrl+/
全局代码格式化（根据语言） Alt+shift+F
选中代码格式化 Ctrl+F

快速跳转

作用 快捷键
快速打开文件 Ctrl+P
快速打开最近文件 Ctrl+R
快速跳转到行（比如报错在第n行） Ctrl+G+行号
快速打开文件第n行 Ctrl+P，输入 :+行号
跳转到函数定义处/跳转到函数实现处 F12/Ctrl+F12
展现谁引用了函数或类 Shift+F12
代码重构（修改一个函数或者变量名） 光标移到修改处+F2

VScode配置云同步

我们可以将配置云同步，这样的话，当我们换个电脑时，即可将配置一键同步到本地，就不需要重新安装插件了，也不需要重新配置软件。

我们还可以把配置分享其他用户，也可以把其他用户的配置给自己用。

将自己本地的配置云同步到 GitHub：

（1）安装插件 settings-sync。

（2）安装完插件后，在插件里使用 GitHub 账号登录。

（3）登录后在 vscode 的界面中，可以选择一个别人的 gist；也可以忽略掉，然后创建一个属于自己的 gist。

（4）使用快捷键 「Command + Shift + P」，在弹出的命令框中输入 sync，并选择「更新/上传配置」，这样就可以把最新的配置上传到 GitHub。

我们如何将Gist中保存的配置同步到本地呢？

首先将gist ID更改为你想要的ID：

打开命令面板，输入sync，选择高级选线，里面就可以更改你想要的gist ID，然后进行下载配置(shift + ALt + D)
