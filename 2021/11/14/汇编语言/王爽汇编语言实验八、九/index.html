实验八

题目

  分析下面的程序，在运行前思考：这个程序可以正确返回吗？

  运行后思考：为什么是这种结果？

  通过这个程序加深对相关内容的理解。

assume cs:codesg
codesg segment
        mov ax,4c00h
        int 21h

start:    mov ax,0
    s:    nop
        nop

        mov di,offset s
        mov si,offset s2
        mov ax,cx:[si]
        mov cs:[di],ax

   s0:    jmp short s
   s1:    mov ax,0
           int 21h
           mov ax,0

   s2:    jmp short s1
           nop
codesg ends
end start

分析

  这个程序究竟干了啥？
  从di,offset s到mov cs:[di],ax这一段干的事就是把s2标号处的代码复制到s标号处，然后下面就是一系列跳转。

  在实际操作之前，我们先简单复习一下：上面这个程序所使用的jmp指令都是短转移jmp short 标号，意味着这些jmp指令不会实际给出要转移到的目的地址，而是给出要转移的位移量。那么这个位移量是怎么算的呢？如下图所示：

解答

  我们首先用debug的-u命令查看反汇编格式：

  观察一下076A:0016和076A:0020的两处jmp短转移指令，分别对应了s0: jmp short s和s2: jmp short s1,它们两的机器码分别是EBF0和EBF6。我们来验证一下短转移指令是不是上面所说的原理：

s0: jmp short s这条指令是要跳转到s处，s标号处所在的地址是076A:0008，而s0:jmp short s这条指令所在的地址是076A:0016,根据上面所说的转移位移的计算方法：s0:jmp short s下一条指令的地址是076A:0018,而目标地址是s标号处所在的076A:0008，两点的距离为08H-18H = -10H （十进制为-16）,其补码表示为F0。s0: jmp short s这条指令显示的机器码是EBF0，可以看到和我们刚刚算出来的F0是一样的。

s2: jmp short s1也是一个道理，就不细说了。

  现在我们开始执行程序，注意由于start标号是在076A:0005,所以程序执行是从这个地址开始，而不是在076A:0000。在s0之前的这段代码做的事情就是把s2标号处的代码复制到s处，我们可以看看执行到s0标号这一步的反汇编格式：

  我们可以看到s标号处（076A:0008）原本的nop指令被s2标号处的EBF6给取代了。这里就是问题的关键，注意短转移指令只提供要转移的位移，而不是目的地址，EBF6的含义是：转移到与当前这个jmp指令的下一条指令距离为F6,也就是-A。

  现在我们继续执行代码，现在CS:IP应该指向s0处，执行后，跳转到标号s处，这个时候就开始执行EBF6了,首先IP+=2，IP = 001A，然后指令缓冲器开始执行EBF6，向前跳转-A,到076A：0000,也就是mov ax,4c00处，下面程序就可以正常返回了。我们来验证一下：

  可以看到现在已经跳转到s标号处了，指令已经显示为JMP 0000。我们再执行一次:

  程序已经开始返回了，说明我们之前说的是对的。实验8到这也就结束了。

分析

assume cs:codesg,ds:data
data segment
        dw 0277H,0265H,026cH,0263H,026fH,026dH,0265H,0220H,0274H,026fH,0220H,026dH,0261H,0273H,026dh,0221H
        dw 2477h,2465h,246ch,2463h,246fh,246dh,2465h,2420h,2474h,246fh,2420h,246dh,2461h,2473h,246dh,2421h
        dw 7177h,7165h,716ch,7163h,716fh,716dh,7165h,7120h,7174h,716fh,7120h,716dh,7161h,7173h,716dh,7121h
data ends

codesg segment
 start: mov ax,data
        mov ds,ax
        mov ax,0B800h
        mov es,ax
        mov bx,0640h
        mov dx,4
        mov di,0

    s0: dec dx
        mov cx,dx
        jcxz stop
        add bx,0a0h
        mov si,0
        mov cx,16

     s: mov ax,[di]
        add di,2
        mov es:[bx+3eh+si],ax
        add si,2
        loop s
        jcxz s0

  stop: mov ax,4c00h
        int 21h

codesg ends
end start
