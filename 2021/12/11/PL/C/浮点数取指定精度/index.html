浮点数取指定精度

发表于2021-12-11|更新于2022-02-13|PLC
|字数总计:778|阅读量:

前言

今天遇到个小问题，就是运算结果需要指定精度的浮点数，比如运算结果必须保留3位小数。如果不用现成的库函数，请问怎么实现？
  这个问题虽小，但是的确是我之前没有想过的。

解决方法

我们先来观察一下一个double型的浮点数强制转换成int型后会输出什么:

可以发现不管小数点后一位是>5还是<5都是直接舍弃取整。如果把double型的数据赋值给int型的数据，结果也是一样的：

如果我们想让强制转换的结果是四舍五入后的结果怎么办？比如说将123.456转换成123,将123.756转换成124。
  很简单，我们加个0.5就行了,这样x.0~x.4的数据就变成了x.5~x.9,强制转化后还是x;x.5~x.9的数据会变成(x+1).0~(x+1).4，强制转化后就变成了x+1，这就达到了进位的效果。

  知道这个方法后，我们想想该怎么利用这一特性来解决我们的问题：将浮点数四舍五入后保留到小数点第三位。

  举个例子，假如我们现在有个double类型的数123.45676，要求四舍五入保留小数点后3位，也就是最后的结果是123.457000000。为了说清楚这个问题，我们先上代码，在来说明：

double x = 123.45676;
int temp;
temp = (x + 0.0005)*1000;    //x+0.0005=123.45726,*1000后就是123457.26，因为是赋值给int型变量，所以0.26被舍去，最后temp=123457
x = temp/1000.0;            //x = 123.457

  可以看到我们完成了将浮点数经过四舍五入后保留到了小数点后3位，这个过程是怎样的呢？我们来描述一下。

x为double型，值为123.45676。我们首先让它+了0.0005，(这在之前讲过了，是为了四舍五入进位)，此时x = 123.45726。然后我们将它×1000然后赋值给一个int变量temp，这样小数点第3位之后的数据就会被舍弃掉，此时temp=123457，我们要的答案是123.457，显然再除个1000赋值给double型变量就是我们要的答案了。

  但是我们注意第4行为什么是x = temp/1000.0,x = temp/1000可以吗？我们可以实际运行一下:

  显然/1000的结果是错的，因为/1000是把这个表达式当成了int型的计算；而/1000.0是把这个表达式当作浮点数的运算，要注意这个区别。

总结

  现在我们知道了将浮点数经过四舍五入转化成指定位数的方法了，我们来稍微总结一下：

  假设我们现在有一个n位的浮点数x,要将它经过四舍五入保留到m位（m<=n），那么我们就可以使用下列步骤:

  1. x = x + 5.0/(m+1)
  2. int temp = x*（10^m）
  3. x = temp / （10^m）.0
