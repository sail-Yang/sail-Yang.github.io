leetcode刷题日记1:无重复字符的最长子串

发表于2022-02-22|更新于2022-02-23|算法刷题日记
|字数总计:1.1k|阅读量:

  作为算法的初学者，为了更好地学习算法，我准备每天都刷一道算法题，然后用日记的方式记录下来，每个周末总结一下这周的日记，提炼出这周题目的算法。

  希望我能坚持下去🤪

  这个是第三题，题目如下：

  这道题显然关键在于两个方面：

  1. 每次怎么切子串。

  2. 怎么判断子串是否重复。

怎么切子串

自己的思路

  对于切子串这个方面，我的想法是用x,y来标识子串的左端和右端，然后对y循环:

  • 每次循环开始用s.substr(x,y)切一个子串。

  • 如果这个子串没有重复字符，记录下长度与目前最大长度比较，如果大于就修改最大长度，然后循环结束，递增y。（右端往右移动）

  • 如果这个子串有重复字符，那么x往右移，y不动。

  上述方法是我一开始做这道题的思路，后来我看了一下题解，发现和双指针滑动窗口算法类似。所以下面说一下双指针滑动窗口算法的思路。

滑动窗口算法

  所谓的窗口就是用一个左指针和右指针来从序列中切出一个子序列，我们就聚焦于这个子序列，就像从一个窗口里看一样，这就是窗口的意思。简单地画个图表示一下：

  那所谓滑动是什么意思呢？

  无非就是左指针往右移动或者右指针往右移动。至于移动哪个就需要结合题目，在移动时我们要思考几个问题:

  1. 满足什么条件时，我们移动right来扩大窗口？又满足什么条件时，我们移动left来缩小窗口呢？

  2. 窗口变动的时候，我们要更新什么数据呢？

  3. 数据是在窗口变动前更新还是在变动后更新呢？

  就比如这个leetcode这道题目，我们会做下面的回答：

  1. 当窗口中无重复子串时，我们移动right；反之移动left。

  2. 在窗口中无重复子串时，我们需要更新最大无重复子串的长度。

  3. 在窗口变动前更新。

  下面是滑动窗口算法的模板：

//窗口初始化
int left = 0;
int right = 0;
//窗口滑动循环,len一般是数组长度或者字符串长度
while(right < s.len) {
    if(isValid()) {
        right++;//窗口扩张
    } else {
        left++;//窗口收缩
    }
}

判断子串是否重复

  实现的方法很多，但我想了一种效率很低的方法…我总结一下我犯的错:

  1. 专门定义了一个函数来判断，这会造成资源的浪费；我看到现在，还没看到专门定义一个函数的。算法题讲究效率，专门定义一个函数讲究的是复用性和可维护性。

  2. 我的主要想法是用map记录子串各字符的个数，如果有>1的就说明重复了。第一个问题是由于原因1，对于每个子串都要重新变量来初始化map，可是明明可以复用这个map的（去掉left指向的字符就行了）；第二个问题是用map的效率真的很高吗？

  由于刚学C++，也只知道map和set😅,自然而然想到了用map来判断的方法。经过学习后，我发现了许多效率更高的方法，下面列举一个。

哈希表

  我们可以用数组来实现一个简单的哈希表int index[128]，映射就是字符的ASCII值。

  • 每次窗口扩大的时候，哈希表中相对应位置+1即可,比如index[s[right]]++。

  • 窗口缩小的时候，哈希表中相对应位置-1即可，比如index[s[left]]–。

  • 如果当前字符在哈希表中的映射位置值不为0，那么就说明重复了。

  差不多就是上面这种思路，下面我们来看看解法：

class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        int res = 0;
        int len = s.length();
        int freq[128] = {0};
        int l = 0, r = -1;
        while (l < len) {
            if (r + 1 < len && freq[s[r + 1]] == 0) {
                freq[s[++r]]++;
            } else {
                freq[s[l++]]--;
            }
            res = max(res, r - l + 1);
        }
        return res;
    }
};
