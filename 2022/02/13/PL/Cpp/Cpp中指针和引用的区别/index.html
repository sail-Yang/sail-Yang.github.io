关于C++中指针和引用的一点小总结

发表于2022-02-13|更新于2022-03-30|PLCpp
|字数总计:1.4k|阅读量:

  C和Cpp都有指针，不过Cpp比C多了一个引用(reference)，一般书里都会说引用是变量的别名（alias），但这种定义太过模糊。于是，趁着这个机会聊聊我理解的引用，顺便也说说指针😁。

指针和引用的逻辑概念

  首先有两个关于指针的概念要搞清楚，就是指针的值和指针的地址。指针本身可以看作和内置类型一样的变量，只不过它的值比较特殊—是其它变量的地址，而它本身所在的内存位置也是一个地址。下面这张图就揭示了这张关系：

  指针的概念比较底层，逻辑上和实际的实现是差不多的；而引用我觉得可能是用指针来实现的，所以现在只能描述一下它的逻辑概念：

  教科书中总说引用就是变量的别名，也就是一个内存块的别名，比如说我定义了一个int型变量:int num = 6，那么再定义一个对num的引用:int &ref = num，那么ref和num逻辑上是高度类似的，就是可以拿ref当num用。

管中窥豹

  上面说了引用的逻辑概念，但我还是好奇引用到底是什么东东。现在有一段代码如下：

int main()
{
    int num1 = 2;
    int *pointer_num1 = &num1;
    int &ref_num1 = num1;
}

  下面我们把这段代码转化成汇编代码：

下图使用godbolt.org这个网站来查看C++代码的x86-64汇编代码，这个网站还能查看每句对应的汇编，十分方便。

  可以看到，引用和指针的汇编级的行为是极度类似的，这至少可以说明引用是以指针为基础来实现的，所以引用也是占内存的。

  但我们知道引用的行为是和指针有差异的，比如引用必须绑定一个变量，之后就不能修改了，而指针可以修改，这种差异又体现在哪呢？

  我觉得差异可能在编译器这一层，机器代码这一层是不大看得出来的。至于引用的机制在编译层的实现我是不太知道的，以后知道了再来填坑吧。

  这种管中窥豹可能并没有什么太大的作用，汇编层只能让我们了解引用本质到底是什么，但是却无法使用汇编代码去解释引用的概念和机制。

引用传递和指针传递

  首先，咱们都知道函数的调用实际上会在栈中开辟一段空间用于存放局部变量，而一般局部变量的生命周期都随着这个被调用函数的结束而Over了。而作为与外界交流的接口的实参，实际上也是局部变量，说到实参就不得不提参数的传递。

  最常见的当然是值传递，其实就是把形参的值拷贝给对应的实参，你这实参怎么变都和主调用函数的形参不搭嘎；如果我们想改变形参的值，在C语言中就得使用指针传递，指针传递实际上也是值传递，只不过指针这种变量的值比较特殊，前面也说了，是其它变量的地址，你这怎么改变实参，还是不会影响主调用函数的形参，不过你可以解引用这个实参，获得对其指向变量的访问，那么就可以修改这个变量了。

  我写了个简单的demo来说明这个问题:

void func(int *ptr){
    *ptr = 10;
    ptr = (int*) 0x61ff1c;
}
int main()
{
    int n = 5;
    int *ptr = &n;
    cout << "ptr's value: "<<ptr << endl;
    cout << "n's value: " << *ptr << endl;
    func(ptr);
    cout << "ptr's value: "<<ptr << endl;
    cout << "n's value: " << *ptr << endl;
}

  那引用传递呢？引用传递就不能叫值传递了，其行为和前两种有点不同，它实际传给局部变量的也是地址，只不过是其引用对象的地址，这就意味着我们不需要解引用这种操作，直接就可以用来改变形参的值。

  如果我们去看编译后文件的符号表，可以看到：

  • 指针变量在符号表上对应的地址值为指针变量的地址值。

  • 引用在符号表上对应的地址值为引用对象的地址值。

拷贝构造函数和引用传递

  现在来想一个小问题：为什么拷贝构造函数的形参是const class &ref，而不是const class reforconst class *ref？

  首先来说说为什么不是值传递，我们在前面解释了值传递会在栈中新建一个局部变量，然后将值拷贝给这个局部变量，而这种传值的方式又会调用这个局部变量的拷贝构造函数，这就会陷入一种对拷贝构造函数的无穷递归。差不多类似于这种概念:

void func(class stu)
{
    func(stu);
}

  再来说说指针传递。首先要说的是参数是指针是不会报错的，只不过这个时候的构造函数就不是拷贝构造函数了，拷贝初始化（就是=）的时候是不会调用这个函数的。那么为什么不把拷贝构造函数的参数设计成指针呢？
