C++虚函数功能及其实现(vptr和vtable)

发表于2022-05-13|更新于2022-05-13|PLCpp
|字数总计:1.1k|阅读量:

虚函数的功能与使用

  C++与C的最大区别就在于C++是面向对象的，而说到面向对象，就绕不开继承、封装和多态。而C++实现多态，靠的就是虚函数。

  首先来看一段来自《C++primer》的话，通过这段话可以对虚函数和多态做一个简单了解:

  当我们使用基类的引用或指针调用基类中定义的一个函数时，我们并不知道该函数的真正作用是什么类型，因为它可能是一个基类的对象也可能是一个派生类的对象。

  如果该函数是虚函数，则直到运行时才会决定执行哪个版本，依据是引用或指针绑定的对象的真正类型。

  也就是说，指向派生类的基类指针可以调用派生类的虚函数。我们来看个例子，假设现在声明了下面这两个类:

class Base {
public:
    Base() = default;
    virtual void func1() {cout << "Base 's func1" << endl; }
    virtual void func2() {cout << "Base 's func2" << endl; }
    ~Base() {}
};

class Child : public Base{
public:
    Child() = default;
    virtual void func1() {cout << "Child's func1" <<endl;} 
    virtual void func2(int i) {cout << "Child's func2" <<endl;}
    ~Child();
    return 0;
}

  这里Base的派生类Child的func2函数故意改了函数签名，这是为了说明一件事，至于是什么，之后会说。我们现在再编写下面这样的main函数测试一下:

int main()
{
    Base *Base_Ptr = new Child();
    Base_Ptr->func1();
    Base_Ptr->func2();
    Base_Ptr = new Base();
    Base_Ptr->func2();
    return 0;
}

  函数的输出如下:

  第一行输出就验证了之前的说法：指向派生类的基类指针可以调用派生类的虚函数，Base_Ptr是指向基类Base的指针，而其指向了Base派生类Child，当调用虚函数func1()时，就会调用派生类Child的版本。我们可以将第一行输出和第三行输出进行一个对比。

  那么按照这个道理来说，第二行输出不应该是调用派生类Child的func2的结果吗？为什么调用了基类Base的func2？

  因为派生类的func2和基类的func2函数签名不一样，前者并不是后者的重写，而是新的虚函数，所以此时只能调用基类的func2。因此，建议在要重写的虚函数处加overrride，这样如果签名不一样，就会报错：

  有了对虚函数基础的印象之后，接下来，我们就来看看虚函数这种机制是如何实现的。

虚函数的实现:vptr 和 vtable

  上面说过我们直到运行时才会决定执行哪个类的虚函数，这种技术就叫作晚捆绑技术（late binding）。它是通过一种叫虚拟表(vtable)的特殊形式来实现的，每个有虚函数的类，都有自己的vtable。

  vtable实际上是一个静态数组，里面包含了该类每个虚函数的地址。编译器还会生成一个隐藏的指向基类的指针VPTR，其在创建类实例（对象）时自动设置，然后指向该类的vtable，所以我们只要一个VPTR就好了，不管这个类有多少个虚函数。示意图如下：

  当我们执行Base Base2 = new Child()时，编译器首先会调用基类的构造函数，这个时候Base2对象的VPTR会首先指向基类Base的Vtable，然后调用派生类的构造函数，这个时候VPTR才会指向派生类Child的基表。

  还有一点需要注意的是虚表可以继承，如果子类没有重写虚函数，那么子类虚表中仍然会有该函数的地址，只不过这个地址指向的是基类的虚函数实现。如果基类有3个虚函数，那么基类的虚表中就有三项（虚函数地址），派生类虚表也会至少有三项，如果重写了相应的虚函数，那么虚表中的地址就会改变，指向自身的虚函数实现。

  如果派生类有自己的虚函数，那么虚表中就会添加该项。就比如我们上面在派生类Child里编写的func2，就是派生类自己的虚函数，其就会被添加进Child的Vtable。
