C++中指针和引用的区别

发表于 2022-02-13 分类于 C/C++ Valine：

  C和C都有指针，不过C比C多了一个引用(reference)，一般书里都会说引用是变量的别名（alias），但这种定义太过模糊。

  下面就来说说C++中的引用到底是什么，以及它和指针有什么区别。

引用是什么

引用的确是已经存在的变量的别名，但别名这种说法太过笼统。事实上，引用就是一个常量指针。为了证明这一点，我们进行一个小实验：

  现在有一段代码：

int main()
{
    int num1 = 2;
    int *pointer_num1 = &num1;
    int &ref_num1 = num1;
}

  下面我们把这段代码转化成汇编代码：

我们会使用godbolt.org这个网站来查看C++代码的x86-64汇编代码，这个网站还能查看每句对应的汇编，十分方便。

  我已经把对应的代码用不同颜色的笔标出来了，可以很清楚地看到指针和引用的行为是一模一样的，这充分说明了指针和引用的基本原理是一样的。

  既然引用和指针一样，那么怎么体现是const指针呢，这汇编代码里也看不出来啊。

  首先要说明，像const这种存取的控制是实现在编译器这一层的，也就是C++底层的机制，在汇编代码这一层是不存在什么const的（我们可以用内存级工具修改任何一个变量）。

  在C++中，引用一旦绑定初始化对象之后，是不能修改其绑定对象的，这和const指针的行为是一模一样的。

  引用只是一种抽象，一种概念，具体怎么实现，可能每个编译器都不一样。上面的汇编只是用来了解引用到底是什么东西，但是不能用汇编代码去解释引用！

内存越界引用和缓冲区溢出问题

发表于 2022-01-24 分类于 C/C++ Valine：

前言

  今天在看CSAPP 3.10节的时候看到了一个熟悉的问题:

如果gets()函数读入的字符数超过了字符数组的界的时候，会发生什么呢？

  我在昨天整理以前的C语言笔记的时候刚刚好试验了一下，试验的过程如下：

假如我执行下面这段代码:

int main()
{
    char str[6];
    gets(str);
    puts(str);
}

  分别输入下面这些数据:

abcde
abcdef
abcdefghijk
abcdefghijklmnopq

  puts输出的都是全的，说明C的gets()和puts()根本就没有检查机制，gets()写入的字符串肯定覆盖了str后面的内容。

  昨天也就调试了一下，并没有细想，比如gets()的实现是什么，覆盖的内容会是什么？会不会造成什么严重的后果？

  今天看CSAPP的时候刚好看到差不多的内容，并且作者指出这种越界问题会导致缓冲区溢出，有许多病毒就是利用缓冲区溢出进行攻击的。

  所以今天想好好思考思考这个问题。

  在这里插一句，最好不要使用gets()，而是使用fgets(),其函数原型为：char * fgets(char *_Buffer, int _MaxCount, FILE *_Stream)。

  连gcc都会提示这么做🤣:

缓冲区溢出

 我们首先来简单看下C语言过程调用中的通用栈帧结构（详细内容见CSAPP）:

  现在有这样一段程序:

#include <stdio.h>
void echo()
{
	char str[6];
	gets(str);
	puts(str);
}

int main()
{
	echo();
	return 0;
}

  我们用objdump -d来反汇编一下，看看程序给gets() 分配了多少空间:

  sub $0x10,%rsp告诉我们，分配了16个字节的空间。用CSAPP中的方式，我们可以列出安全区域和危险区域：

  也就是说，只要输入15个字符以内，不会有严重后果，但是如果超过了，没有保护机制的话，存储的返回地址就会被破坏。返回的时候就会跳转到一个莫名其妙的地方。

缓冲区溢出的攻击与对抗（全部来自CSAPP）

  一些病毒就使用缓冲区溢出来让程序执行它本来不愿意执行的函数。比如输入给程序一个字符串，这个字符串中包含了可执行代码的字节编码（被称为exploit code），另外，还有一些字节会用一个指向攻击代码的指针覆盖返回地址。那么，执行ret就会跳转到攻击代码。

比如说1988.11的Internet蠕虫病毒（详细见CSAPP）

  下面重点介绍GCC对于缓冲区溢出的保护机制。

栈随机化

  前面提到了攻击者除了注入exploit code，还要插入指向这段code的指针，这显然需要知道存放这个字符串的栈地址。之所以能够实现，就是因为以前栈地址是非常容易预测的，在不同机器里，栈的位置都差不多。这也是为什么1988年的蠕虫病毒只要确定一个Web服务器所使用的栈空间，就能对许多机器进行攻击。这种现象叫作安全单一化。

  栈随机化的思想就是使得栈的位置在每次程序运行的时候都有变化。其实现方法是：

  程序开始时在栈上分配随机的空间，然后程序使用这段空间之后的空间，这样就能起到随机的作用。

  在64位Linux上，不断运行下面这个程序，就能看到这种栈地址变化:

int main()
{
    long local;
    printf("local at %p\n",&local);
    return 0;
}

这种方法属于更大的一类技术，这类技术叫ASLR（地址空间布局随机化）。但是也有办法对付ASLR，比如一种操作叫nop sled.

栈破坏检测

  GCC在产生的代码中加入了一种机制来检测缓冲区越界，其思想也很简单，就是在局部缓冲区和状态区之间放一个哨兵值（不知道为什么叫canary,金丝雀😅），这个是随机产生的。

  在恢复寄存器状态和返回之前，程序会检查这个值是否被改变了，如果是，就直接中止。如果我们想取消这种检测，可以加参数-fno-stack-protector。

对书后习题的态度

发表于 2022-01-22 分类于 Let's Talk Valine：

  我接受信息的方式主要是看文章和书，而专业书中每一章结尾都有习题或者实验，之前我的学习方式都是看完每章的正文内容之后就写个笔记，然后就直接跳到下一章了。我现在觉得这种方式的学习效率十分低下，因为整个过程几乎都是被动地接受信息，学习效果不是很好。
  理想的学习方式当然是学完一章的内容后，去用本章知识解决问题，达到巩固知识的作用，除此之外还可以根据本章主题拓展一下。

  我之前也是这么想的，可是为什么没这么做呢？

  1. 许多书的习题是不在书中提供的，而是要到资源网站上下载，我老是懒得去找。

  2. 大多数专业书习题特别多，还难，我有种畏难心理，并且觉得没啥用。

  3. 专业书每章的信息量特别大，看完正文内容就没心思写题目了。我又喜欢多线程操作，觉得没有必要花太多时间在习题上。

  首先要做的就是认识到习题的重要性，学习计算机和数学最好的方式就是实践，习题又是作者故意设置的有针对性的实践，所以是值得一做的。其次，接下来每次开始给自己造坑时，都把习题答案准备好：

  1. 不在书里的去官网上下载到云盘里，如果不提供去网上找找。

  2. 在书里的，做好PDF的标签。

  正文内容看完可能会觉得很累，可以另一个时间段再写题目，这样能起到一个回顾与巩固的作用。

  我看书有点太过被动，应该更加灵活一点，特别是要注意下面几点：

  不仅要知道这个知识是什么，还要会用这个知识去解决问题。

  解决某个具体问题的方法可能不是最重要的，而是解决这类问题的方法。

有符号数转为无符号数

  我们都知道像double x = 5;``int y = 3.14;都会自动进行类型转换，那么如果把一个负数(int型)赋值给一个unsigned类型的无符号数，会进行怎样的类型转换呢？我们来看一看下面这个例子：

int i = -42;
unsigned u = i;
cout << dec << "u的十进制结果为：" << u << endl;
cout << "u的十六进制结果为: " << hex << u << endl;
cout << "i的十六进制结果为：" << hex << i << endl;

  输出的结果为：

  由此可见，有符号数转为无符号数就是把原本的补码表示转化为原码表示。

  值得注意的是：算术运算中，如果有符号数和无符号数同时存在，有符号数会自动类型转化为无符号数。我们来看《C++primer》的一个例子：

int i = -42;
unsigned u = 10;
cout << u + i << endl;
cout << i + i << endl;

  u+i中u 是无符号数，i是有符号数，两者相加时，首先会把i转化为无符号数，也就是4294967254，然后再+u，最后结果应该是4294967264；i+i都是有符号数，就是正常的有符号数加法，结果应该是-84，下面来看一下程序的运行结果：

此问题相关的bug

  上文其实告诉我们在算术运算中，只要式子里有无符号数，一定要保证结果不是负的。如果不这么做，就会产生bug，比如下面这个例子（来自《C++Primer》）:

for(unsigned u = 10;u>=0;--u)
    cout << u << endl;

  这段看上去是个很普通的for循环，但是这个程序是无限死循环的，为什么呢？

  很简单，当u=0之后，如果再--u ，u就要进行类型转换了，自动把-1转化为无符号数4294967295,然后这就会继续循环下去。

把超出范围的值赋值给无符号数

  如果我们把一个超出无符号数范围的值赋给无符号数，会发生什么呢？

  当我们赋给无符号类型一个超出它范围的值时，结果是初始值对无符号类型表示范围总数取模后的余数。

  这个也很好理解，我们来举个例子，现在有一个unsigned char类型的变量为ch,它的表示范围为0~255，范围总数也就是256，我们看看现在赋不同值给它的结果如何：

unsigned char ch = 256;
cout << dec << (unsigned) ch << endl;    //256 % 256 = 0
ch = 257;
cout << dec << (unsigned) ch << endl;    //257 % 256 = 1
ch = 258;
cout << dec << (unsigned) ch << endl;    //258 % 256 = 2

  程序的运行结果如下：

  上面的例子都是正数，如果是负数呢？我们分别把-1,-2,-3赋值给ch，最后的结果是255,254,253 。其实这也就是-1 % 256 = -1、-2 % 256 = -2、-3 % 256=-3的无符号数结果。

  关于负数取模问题，我们可以记住两个公式：

  1. a mod b = a mod (-b)

  2. a mod b = - (-a mod b)

把超出范围的值赋值给有符号数

  就是溢出，会导致未定义的结果。所以一定要熟记基本类型的数据范围：

//查看基本数据类型范围
#include <iostream>
#include <float.h>//里面有float类型的最大值和最小值

using namespace std;
int main()
{
    char ch = 129;
    cout << "int型\t\t"<<sizeof(int)<<"字节\t"<<sizeof(int)*8 << "位\t"<< INT_MIN<<"~"<<INT_MAX<< "\n" << endl;
    cout << "long型\t\t"<<sizeof(long)<<"字节\t"<<sizeof(long)*8 << "位\t"<< LONG_MIN<<"~"<<LONG_MAX<< "\n" << endl;
    cout << "long long型\t"<<sizeof(long long)<<"字节\t"<<sizeof(long long)*8 << "位\t"<< LONG_LONG_MIN<<"~"<<LONG_LONG_MAX << "\n" << endl;
    cout << "char型\t\t"<<sizeof(char)<<"字节\t"<<sizeof(char)*8 << "位\t"<< CHAR_MIN<<"~"<<CHAR_MAX<< "\n" << endl;
    cout << "float型\t\t"<<sizeof(float)<<"字节\t"<<sizeof(float)*8 << "位\t"<< FLT_MIN<<"~"<<FLT_MAX<< "\n" << endl;
    cout << "double型\t"<<sizeof(double)<<"字节\t"<<sizeof(double)*8 << "位\t"<< DBL_MIN<<"~"<<DBL_MAX<< "\n" << endl;
